NWS Weather  


Representational State Transfer (REST) allows two computers to transfer data using only HTTP protocols. [27] [55] [57] It first appeared in 2000. [56] REST APIs support highly performant and reliable secure communication at scale. They are flexible and cross-platform portable. They are easy to implement and to modify. [52] Simple Object Access Protocol (SOAP) was previously used to transfer data, but has mostly been replaced by REST. REST uses less bandwidth and is simpler. JSON is the most popular data format used with REST. [55] [57]

This application provides a demo of using REST to retrieve an extended weather forecast from the National Weather Service (NWS) using their public REST API. [43] [44] Select a location from the drop-down list and click the NWS button. This will initiate a two step process. Step One will query the API to determine if a forecast exists for the desired location. If a forecast exists, Step Two will download the data and display it on the computer screen. Forecasts are arranged based on the NWS office responsible for that area, so the city in the forecast may not always match the city selected in the drop-down list. If you know the coordinates, you can enter them directly like this:

39.67,-97.08

Do not enter a space after the comma, or the API will respond with an error. If you enter a location which is not in the database, the response will be error 404 (not found). You can see that by selecting either Paris, France or Toronto, ON from the drop-down list. If the response is error 500 (internal server error), please wait a moment for the condition to clear before repeating the request.
